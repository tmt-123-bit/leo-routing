# 实验设计

## 实验目标

验证基于 PS-MAPPO 的队列感知与链路寿命约束分布式动态路由方法是否能够在 LEO 高动态网络中实现：

- 更低的端到端时延。
- 更低的队列积压和链路拥塞。
- 更高的投递成功率和吞吐量。
- 更少的链路失效重路由和路由震荡。
- 更低的控制开销。

## 仿真系统模型

LEO 卫星网络建模为时变图：

```text
G(t) = (V, E(t), X(t))
```

其中：

- `V`：卫星节点集合。
- `E(t)`：时刻 `t` 的可用星间链路。
- `X(t)`：节点和链路状态，包括队列、带宽、传播时延、链路负载、可靠性和链路剩余可用时间。

每颗卫星是一个智能体，执行阶段只使用局部观测和一跳邻居信息。

## 实验场景

### 场景 1：低负载场景

目的：

验证本文方法在轻负载下不会明显劣于最短路径算法。

预期：

- Dijkstra 可能有较低时延。
- 本文方法因为考虑稳定性和拥塞预防，时延可能接近 Dijkstra。
- 队列感知优势不明显。

### 场景 2：高负载热点场景

目的：

验证队列感知和链路负载感知能否缓解热点拥塞。

设置：

- 部分地面站或目的区域产生突发流量。
- 部分卫星成为热点转发节点。

预期：

- Dijkstra 和最短路径方法容易把流量集中到少数链路。
- 本文方法应降低平均队列长度、最大队列长度和丢包率。
- 本文方法应提高吞吐量和投递成功率。

### 场景 3：链路频繁断开场景

目的：

验证链路剩余可用时间约束是否有效。

设置：

- 增加跨轨链路断开频率。
- 设置极区或反向缝附近链路变化。

预期：

- 不考虑链路寿命的方法会选择即将断开的链路。
- 本文方法应减少路径中断次数、重路由次数和丢包率。
- 端到端时延曲线应更稳定。

### 场景 4：节点或链路故障场景

目的：

验证方法的鲁棒性和恢复能力。

设置：

- 随机关闭部分链路。
- 随机设置部分链路高 `P_out`。

预期：

- 本文方法应更快避开故障链路。
- 故障恢复时间更短。
- 投递成功率更高。

### 场景 5：不同星座规模场景

目的：

验证参数共享 PS-MAPPO 的泛化能力。

设置：

- 小规模星座。
- 中等规模星座。
- 较大规模星座。

预期：

- 参数共享模型应具有较好的可扩展性。
- 相比每个卫星独立模型，存储开销明显更低。

## 对比算法

建议至少选择以下算法：

1. Dijkstra 最短路径。
2. 虚拟拓扑最短路径路由。
3. Q-routing 或 QRLSN。
4. 普通 MAPPO，不加入队列和链路寿命特征。
5. 本文完整 PS-MAPPO 方法。

如果时间有限，最小对比组合可以是：

```text
Dijkstra + Q-routing + MAPPO baseline + proposed PS-MAPPO
```

## 消融实验

消融实验是证明每个设计因素有用的关键。

建议设置：

1. 完整方法。
2. 去掉队列惩罚 `-q_j`。
3. 去掉链路负载项 `-rho_ij`。
4. 去掉链路剩余可用时间项 `T_rem`。
5. 去掉 action mask。
6. 去掉全局协同奖励，只使用局部奖励。
7. 去掉路由切换惩罚。
8. 去掉环路惩罚。

预期现象：

- 去掉队列项：高负载场景下队列增长，时延上升。
- 去掉负载项：热点链路利用率升高，负载不均衡。
- 去掉链路寿命项：路径中断和重路由次数增加。
- 去掉 action mask：可能选到不可用或高风险链路。
- 去掉全局奖励：智能体局部最优增强，全网拥塞更严重。
- 去掉切换惩罚：路由震荡和切换次数增加。
- 去掉环路惩罚：平均跳数、TTL 丢包和无效转发增加。

## 核心评价指标

### 时延指标

- 平均端到端时延。
- 95% 分位端到端时延。
- 平均排队时延。
- 平均传输时延。

### 吞吐与可靠性指标

- 网络总吞吐量。
- 投递成功率。
- 丢包率。
- TTL 超时丢包率。

### 拥塞和负载指标

- 平均队列长度。
- 最大队列长度。
- 链路负载方差。
- 节点负载方差。
- 最大链路利用率。

### 稳定性指标

- 路由切换次数。
- 路由震荡次数。
- 路径中断次数。
- 链路断开后的恢复时间。

### 控制与计算开销

- Hello 包数量。
- 控制消息字节数。
- 单次路由决策时间。
- 模型参数量。
- 训练收敛轮数。

## Hello 周期实验

设置不同邻居状态交换周期：

```text
0.5 s, 1 s, 2 s, 5 s
```

观察：

- 控制开销。
- 平均端到端时延。
- 丢包率。
- 路由切换次数。

预期：

周期越短，状态越新，路由更准确，但控制开销越高。周期越长，控制开销越低，但状态过期可能导致错误选择。

## 奖励权重敏感性实验

测试不同权重对结果的影响：

- 增大队列权重 `w_queue`。
- 增大链路寿命权重 `w_lifetime`。
- 增大切换惩罚 `w_switch`。
- 增大吞吐奖励 `w_throughput`。

预期：

- 队列权重过小：拥塞缓解不明显。
- 队列权重过大：绕路增加。
- 链路寿命权重过小：仍可能选到快断链路。
- 链路寿命权重过大：路径过于保守。
- 切换惩罚过小：路由震荡。
- 切换惩罚过大：拥塞路径切换不及时。

## 最终要证明的结论

本文实验最终要证明：

**PS-MAPPO + 队列感知 + 链路寿命约束 + 动作掩码** 能够在 LEO 高动态网络中比传统最短路径、Q-routing 和普通 MAPPO 更好地兼顾低时延、抗拥塞、路径稳定性和低控制开销。
